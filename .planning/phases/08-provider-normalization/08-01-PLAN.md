   - `normalizeCodexUsage(raw)`: Takes `CodexThreadTokenUsage` (with `total` and `last` breakdown objects using camelCase fields: `inputTokens`, `outputTokens`, `cachedInputTokens`, `reasoningOutputTokens`, `totalTokens`, plus optional `modelContextWindow`). Maps `raw.total` fields to `NormalizedTokenUsage`, mapping `reasoningOutputTokens` to `reasoningTokens`.

   - `normalizeClaudeUsage(raw)`: Takes `ClaudeRawUsage` (snake_case fields: `input_tokens`, `output_tokens`, `cache_creation_input_tokens`, `cache_read_input_tokens`). Maps to `NormalizedTokenUsage` with `totalTokens = input_tokens + output_tokens`, `cachedInputTokens = cache_read_input_tokens`. Note: `cache_creation_input_tokens` is NOT mapped -- it's already counted in `input_tokens`.

   - `normalizeGeminiUsage(raw)`: Takes `GeminiRawUsageMetadata` (optional fields: `promptTokenCount`, `candidatesTokenCount`, `totalTokenCount`, `cachedContentTokenCount`, `thoughtsTokenCount`). Defaults undefined numeric fields to 0 for input/output. Uses `totalTokenCount` if available, otherwise computes `input + output`. Only includes `cachedInputTokens` and `reasoningTokens` when present in the raw data.

   All three functions return `NormalizedTokenUsage` (import type from `@xbetools/contracts`). Define the provider-specific raw types as TypeScript interfaces (not Effect schemas -- these are pure input types). Export the three functions and the three raw type interfaces.
