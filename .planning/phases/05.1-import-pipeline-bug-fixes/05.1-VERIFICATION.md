### Summary

All three bugs from the v1.0 milestone audit are closed:

1. **GAP-1 (CRITICAL) — projector providerThreadId propagation:** `projector.ts` line 265 now includes `providerThreadId: payload.providerThreadId ?? null` in the `thread.created` object literal. The `?? null` fallback preserves backward compatibility with events predating the field. Regression test at `projector.test.ts:96-127` proves the value propagates. Downstream consumers in `HistoryMaterializer` (dedup) and `ProviderCommandReactor` (native resume) both read from the in-memory read model and now receive the correct value.

2. **GAP-2 (MEDIUM) — badge prefix mismatch:** `SessionListStep.tsx` `isAlreadyImported` now constructs `${session.providerName}:${id}` before comparing against the `providerThreadIds` Set (which holds prefixed strings). Previously, raw UUIDs were compared against prefixed strings, guaranteeing a miss. The fix uses `session.providerName` from `HistoryImportConversationSummary` which carries the exact provider name used as the prefix in `HistoryImportService`.

3. **Tech debt — HistoryImportValidationStatus schema:** `"importing"` is now in the `Schema.Literals` array. This aligns the contract with what `HistoryMaterializer` writes to the DB (`"importing"` during in-progress import) and what `ProvenanceCard` handles in its switch statement.
