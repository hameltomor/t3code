  <files>packages/contracts/src/orchestration.ts</files>
  <action>
Add the following schemas to `packages/contracts/src/orchestration.ts`, placed AFTER the existing `OrchestrationSession` schema block and BEFORE `OrchestrationThread`:

1. **NormalizedTokenUsage** schema (exported):
```typescript
export const NormalizedTokenUsage = Schema.Struct({
  inputTokens: Schema.optional(NonNegativeInt),
  outputTokens: Schema.optional(NonNegativeInt),
  totalTokens: NonNegativeInt,
  cachedInputTokens: Schema.optional(NonNegativeInt),
  reasoningTokens: Schema.optional(NonNegativeInt),
});
export type NormalizedTokenUsage = typeof NormalizedTokenUsage.Type;
```
- `totalTokens` is required (all providers can supply or compute it)
- All other fields are optional (provider-specific variance -- e.g. `cachedInputTokens` from Codex/Claude but not Gemini, `reasoningTokens` from Codex only)

2. **Four context status enum schemas** (all exported with their types):
- `ContextStatusSupport` = `Schema.Literals(["native", "derived-live", "derived-on-demand", "unsupported"])` -- SDM-03
- `ContextStatusSource` = `Schema.Literals(["provider-event", "sdk-usage", "count-tokens", "heuristic"])` -- SDM-04
- `ContextStatusFreshness` = `Schema.Literals(["live", "stale", "unknown"])` -- SDM-05
- `ContextStatusLevel` = `Schema.Literals(["ok", "watch", "near-limit", "compacted", "unknown"])` -- SDM-06

3. **OrchestrationThreadContextStatus** schema (exported):
```typescript
export const OrchestrationThreadContextStatus = Schema.Struct({
  provider: ProviderKind,
  support: ContextStatusSupport,
  source: ContextStatusSource,
  freshness: ContextStatusFreshness,
  status: ContextStatusLevel,
  model: Schema.NullOr(TrimmedNonEmptyString),
  tokenUsage: Schema.NullOr(NormalizedTokenUsage),
  contextWindowLimit: Schema.optional(NonNegativeInt),
  percent: Schema.optional(Schema.Number),
  lastCompactedAt: Schema.optional(IsoDateTime),
  lastCompactionReason: Schema.optional(TrimmedNonEmptyString),
  compactionCount: Schema.optional(NonNegativeInt),
  measuredAt: IsoDateTime,
});
export type OrchestrationThreadContextStatus = typeof OrchestrationThreadContextStatus.Type;
```
- Compaction history is limited to `lastCompactedAt`, `lastCompactionReason`, `compactionCount` per SDM-07
- `measuredAt` is required per SDM-02

4. **Add `contextStatus` field to `OrchestrationThread`** -- SDM-08:
Add this field to the existing `OrchestrationThread` Schema.Struct, after the `session` field:
```typescript
contextStatus: Schema.NullOr(OrchestrationThreadContextStatus).pipe(
  Schema.withDecodingDefault(() => null),
),
```
This follows the same pattern as `providerThreadId` and `worktreeEntries` -- nullable with decoding default for backward compatibility with existing persisted data.
