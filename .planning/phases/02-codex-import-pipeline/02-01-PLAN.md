**3. Create `apps/server/src/historyImport/Layers/CodexHistoryScanner.ts`:**

Implement the scanner layer. Key responsibilities:

**3a. Determine Codex home directory:**
- Check `process.env.CODEX_SQLITE_HOME` first
- Fall back to `path.join(os.homedir(), ".codex")`
- Allow override via `input.codexHome` parameter

**3b. Open state_5.sqlite read-only:**
Use `NodeSqliteClient.layer` with `{ filename: path.join(codexHome, "state_5.sqlite"), readonly: true }` to create a scoped read-only connection. After acquiring the client, execute `PRAGMA busy_timeout = 5000` for WAL safety.

IMPORTANT: Do NOT use the app's existing SqlClient service for this -- the Codex database is a SEPARATE SQLite file from the XBE database. Create a new scoped connection using `NodeSqliteClient.layer` directly, and provide it locally to the query effect. Use `Effect.provide` to scope the Codex SQL client to just the query, not the full service.

**3c. Query threads table for matching workspace:**
```sql
SELECT id, rollout_path, cwd, title, model_provider, created_at, updated_at,
       source, first_user_message, tokens_used, agent_nickname, cli_version
FROM threads
WHERE (cwd = ? OR cwd LIKE ? || '/%')
  AND source IN ('cli', 'vscode')
  AND agent_nickname IS NULL
  AND archived = 0
ORDER BY updated_at DESC
```

Pass `workspaceRoot` as both `?` parameters.

**3d. Convert timestamps:**
Codex stores `created_at` and `updated_at` as epoch seconds (INTEGER), NOT ISO strings. Convert: `new Date(epochSeconds * 1000).toISOString()`.

**3e. Glob rollout files (validation + fingerprint):**
For each thread row, verify the `rollout_path` file exists. If the thread's `rollout_path` is null or the file doesn't exist, use `tinyglobby.glob` to search `${codexHome}/sessions/**/*.jsonl` filtered by the thread ID in the filename.

Use `import { glob } from "tinyglobby"` with pattern `sessions/**/*.jsonl` and `cwd: codexHome`.

**3f. Compute fingerprint:**
For each rollout file found:
```typescript
import { createHash } from "node:crypto"
import { stat, open } from "node:fs/promises"

async function computeFingerprint(
  sessionId: string,
  filePath: string,
): Promise<string> {
  const fileStat = await stat(filePath)
  const fileSize = fileStat.size
  const mtimeMs = fileStat.mtimeMs

  const SAMPLE_SIZE = 4096
  const handle = await open(filePath, "r")
  try {
    const headBuf = Buffer.alloc(Math.min(SAMPLE_SIZE, fileSize))
    await handle.read(headBuf, 0, headBuf.length, 0)

    let tailBuf = Buffer.alloc(0)
    if (fileSize > SAMPLE_SIZE) {
      tailBuf = Buffer.alloc(Math.min(SAMPLE_SIZE, fileSize))
      await handle.read(tailBuf, 0, tailBuf.length, Math.max(0, fileSize - SAMPLE_SIZE))
    }

    const hash = createHash("sha256")
    hash.update(sessionId)
    hash.update(String(fileSize))
    hash.update(String(mtimeMs))
    hash.update(headBuf)
    if (tailBuf.length > 0) hash.update(tailBuf)
    return hash.digest("hex")
  } finally {
    await handle.close()
  }
}
```

Wrap this in `Effect.tryPromise` with proper error mapping to `HistoryImportScanError`.

**3g. Build catalog entries and upsert:**
For each valid thread+rollout pair, build a `HistoryImportCatalogEntry`:
- `catalogId`: `codex:${threadRow.id}` (namespaced to prevent cross-provider collisions)
- `providerName`: `"codex"`
- `workspaceRoot`: the input workspace root
- `cwd`: `threadRow.cwd`
- `title`: `threadRow.title || threadRow.first_user_message || "Untitled Codex Session"`
- `model`: `threadRow.model_provider || null`
- `messageCount`: 0 (will be populated by preview/import, not scan)
- `turnCount`: 0 (same)
- `providerConversationId`: `threadRow.id`
- `providerSessionId`: `threadRow.id`
- `resumeAnchorId`: `threadRow.id` (used for native resume)
- `sourceKind`: `"codex-rollout"`
- `sourcePath`: absolute path to the rollout file
- `linkMode`: `"native-resume"` (Codex supports native resume)
- `validationStatus`: `"valid"` (just scanned, file exists)
- `warningsJson`: `"[]"`
- `fingerprint`: computed fingerprint
- `rawMetadataJson`: `JSON.stringify({ source: threadRow.source, tokens_used: threadRow.tokens_used, cli_version: threadRow.cli_version })`
- `createdAt`, `updatedAt`, `lastScannedAt`: current ISO datetime

Upsert each entry via `HistoryImportCatalogRepository.upsert()`.

**3h. Error handling:**
- If `state_5.sqlite` doesn't exist, return empty array (not an error -- user may not have Codex installed)
- If a single rollout file fails fingerprinting, log warning and skip it (NFR-4: scan isolation)
- Wrap SQLite errors in `HistoryImportScanError`
