**2. Create `ClaudeCodeSessionParser` layer** with streaming JSONL parsing:

- Acquire `FileSystem.FileSystem` at LAYER CONSTRUCTION TIME (not inside parse method) to prevent context leaking into the return type. This is critical -- see decision from Phase 02-02.
- Streaming approach: `fs.stream(filePath)` -> `Stream.decodeText("utf-8")` -> `Stream.splitLines` -> `Stream.runFold` with mutable accumulator state.
- For each line:
  - Parse JSON, then attempt `Schema.decodeUnknownOption` for each known line type.
  - **Skip lines where** `type` is `"progress"`, `"system"`, `"file-history-snapshot"`, `"queue-operation"`. These are metadata, not conversation content.
  - **Skip lines where** `isSidechain === true`.
  - **For `type: "user"` lines:**
    - Skip if `isMeta === true` (system injections).
    - If `content` is a string: create a message with `role: "user"`, `text: content`.
    - If `content` is an array:
      - Extract `text` blocks -> concatenate into message text.
      - Extract `tool_result` blocks -> create activities with `kind: "tool_result"`, `tone: "tool"`, `summary: "Tool result for <tool_use_id>"`, `payload: { content }`.
    - If user message has `toolUseResult` field (object with stdout/stderr/interrupted): create activity with `kind: "tool_execution"`, `tone: "tool"`, `summary` = truncated stdout (first 200 chars) or "Tool execution" if empty.
  - **For `type: "assistant"` lines:**
    - Track `uuid` as potential `lastAssistantUuid` for resume seed.
    - Iterate `message.content` blocks:
      - `thinking` block -> activity with `kind: "thinking"`, `tone: "info"`, `summary` = truncated `thinking` text (first 200 chars). Do NOT include as message text (FR-5 requirement).
      - `tool_use` block -> activity with `kind: "tool_use"`, `tone: "tool"`, `summary` = `"${name}(${truncatedInput})"` where truncatedInput is first 150 chars of `JSON.stringify(input)`.
      - `text` block -> accumulate into message text.
    - After processing all content blocks: if there is accumulated text, create a message with `role: "assistant"`, `text: accumulatedText`, `turnId: uuid`.
    - Extract `model` from `message.model` for session metadata if not yet set.
  - **Incomplete message detection** (Pitfall 2): A `stop_reason: null` assistant message is incomplete ONLY if it is the absolute last assistant message AND there is no subsequent user message. Mark such messages with `isStreaming: true`. All other `stop_reason: null` messages are valid intermediate responses.
  - Track `turnId` by using the assistant message `uuid` for both the assistant messages and the activities within that turn.
- **Capping**: Honor `maxMessages` and `maxActivities` options. Once capped, continue scanning for metadata (sessionMeta, lastAssistantUuid) but stop accumulating messages/activities.
- **Session metadata**: Extract from first non-skipped line: `sessionId`, `cwd`, `version`, `gitBranch`.
- **Return** `ClaudeCodeParseResult` including `lastAssistantUuid` (the uuid of the last complete assistant message for resume seed).
- Add `Effect.withSpan("ClaudeCodeSessionParser.parse")`.
