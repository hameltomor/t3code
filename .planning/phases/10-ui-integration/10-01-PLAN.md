1. Create `apps/web/src/components/contextStatusIndicator.logic.ts`:
   - Export type `ContextThreshold = "neutral" | "watch" | "warning" | "danger"`.
   - Export interface `ContextStatusDisplay` with fields:
     - `label: string` -- "Context 42%" or "Context unknown" or ""
     - `threshold: ContextThreshold`
     - `visible: boolean` -- whether to render the badge at all
   - Export function `deriveContextThreshold(percent: number | undefined): ContextThreshold`:
     - `undefined` -> "neutral"
     - `>= 95` -> "danger"
     - `>= 85` -> "warning"
     - `>= 70` -> "watch"
     - else -> "neutral"
   - Export function `deriveContextStatusDisplay(contextStatus: OrchestrationThreadContextStatus | null, sessionActive: boolean): ContextStatusDisplay`:
     - If `!sessionActive || !contextStatus` -> `{ label: "", threshold: "neutral", visible: false }`
     - `label`: if `percent !== undefined` then `"Context ${Math.round(percent)}%"` else `"Context unknown"`
     - `threshold`: from `deriveContextThreshold(contextStatus.percent)`
     - `visible: true`
   - Import `OrchestrationThreadContextStatus` type from `@xbetools/contracts`.

2. Create `apps/web/src/components/ContextStatusIndicator.tsx`:
   - Import `Badge` from `./ui/badge`.
   - Import `ContextStatusDisplay`, `ContextThreshold` from `./contextStatusIndicator.logic`.
   - Define a constant mapping `THRESHOLD_BADGE_VARIANT` mapping ContextThreshold to Badge variant:
     - neutral -> "outline"
     - watch -> "info"
     - warning -> "warning"
     - danger -> "error"
   - Export `ContextStatusIndicator` component:
     - Props: `{ display: ContextStatusDisplay }`
     - If `!display.visible` return `null`
     - Render a `<Badge>` with variant from the mapping and size="sm", showing `display.label`.
   - The component is a thin render layer -- all logic lives in the `.logic.ts` file.

3. In `apps/web/src/components/ChatView.tsx`:
   - Import `ContextStatusIndicator` from `./ContextStatusIndicator`.
   - Import `deriveContextStatusDisplay` from `./contextStatusIndicator.logic`.
   - In the `ChatViewInner` component (or wherever the thread and session are available), derive the display state:
     - Determine `sessionActive`: `thread.session !== null && thread.session.status !== "closed" && thread.session.orchestrationStatus !== "stopped"`
     - Call `deriveContextStatusDisplay(thread.contextStatus, sessionActive)` and memoize with `useMemo` depending on `[thread.contextStatus, sessionActive]`.
   - In the composer footer's left-side controls div (after the runtime mode toggle button, before the closing `</div>` at approximately line 4021), add:

     ```jsx
     {contextStatusDisplay.visible && (
       <>
         {!composerCompact && (

       </>
     )}
     ```
   - This placement puts the context status badge as the last item in the left-side controls, after runtime mode and before the right-side actions (attach + send/stop).
